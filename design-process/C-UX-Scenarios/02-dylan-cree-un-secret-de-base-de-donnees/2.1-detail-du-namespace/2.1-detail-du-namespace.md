# 2.1-Détail-du-Namespace

## Page Metadata

| Property | Value |
|----------|-------|
| **Scenario** | Dylan crée un secret de base de données |
| **Page Number** | 2.1 |
| **Platform** | Desktop |

## Overview

**Page Purpose:** Permettre au développeur de visualiser les secrets existants de son namespace et d'en créer de nouveaux.

**User Situation (Q3):** Dylan le Développeur est en train de déployer un nouveau micro-service et doit configurer les accès à sa DB.

**Mental State (Q4):** 
- **Hope:** Pouvoir ajouter son secret rapidement sans erreur.
- **Worry:** Devoir manipuler des fichiers YAML complexes ou attendre un SRE.

**Discovery Method (Q6):** Lien direct "Add Secret" depuis son portail développeur interne.

**Entry Context:** Arrivée directe sur le namespace spécifique (filtré via le lien du portail dev).

**Exit Action:** Dylan clique sur le bouton "Nouveau Secret" (en haut à droite).

**On-Page Interactions:**
- Filtrer la liste des secrets existants.
- Voir l'état de synchronisation des secrets déjà déployés.
