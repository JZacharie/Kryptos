# 2.1 Sélection du Template

**Scenario:** Dylan crée un secret de base de données
**Page Number:** 2.1
**Created:** 2026-04-24
**Method:** Whiteport Design Studio (WDS)

---

## Overview

**Page Purpose:** Aider Dylan à choisir le bon modèle de secret pour son application (ex: PostgreSQL, Redis, API Key).

**Entry Points:**
- Bouton "+ Nouveau Secret" depuis le Dashboard (non encore implémenté sur 1.1, à ajouter).
- Lien direct "Self-Service" dans la barre latérale.

**User Mental State:**
En quête d'efficacité. Dylan est en plein milieu d'un sprint de développement. Il veut que le secret soit créé correctement du premier coup pour ne pas avoir de rejet lors de la review sécurité.

**Main User Goal:** Sélectionner le template "Database Credentials".

**Business Goal:** Standardiser la création de secrets via des modèles approuvés.

**URL/Route:** `/create/template`

---

## Layout Sections

### 1. Wizard Header
- **Purpose:** Indiquer la progression dans le processus de création.
- **Priority:** Secondary.
- **Objects:**
    - `stepper`: Étape 1/3 - Sélection du modèle.

### 2. Template Selector (Cards)
- **Purpose:** Afficher les options disponibles sous forme de cartes visuelles.
- **Priority:** Primary.
- **Objects:**
    - `template-card-db`: Template Base de données (Sélectionné par défaut pour le scénario).
    - `template-card-api`: Template API Key.
    - `template-card-cert`: Template Certificat SSL.

---

## Interactions & States

- **`template-card`:**
    - **Selection:** Clic sur une carte active le bouton "Continuer".
    - **Visual State:** Bordure `accent-secondary` (Magenta) sur la carte sélectionnée.
