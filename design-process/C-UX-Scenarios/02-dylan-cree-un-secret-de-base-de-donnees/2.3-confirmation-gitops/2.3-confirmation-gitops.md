# 2.3 Confirmation et Instructions GitOps

**Scenario:** Dylan crée un secret de base de données
**Page Number:** 2.3
**Created:** 2026-04-24
**Method:** Whiteport Design Studio (WDS)

---

## Overview

**Page Purpose:** Confirmer que le secret a été scellé et que la Pull Request a été créée dans le dépôt GitOps.

**User Mental State:**
Satisfaction et clôture. Dylan a fini sa tâche. Il attend maintenant le lien vers GitHub/GitLab pour taguer son lead pour la review.

**Main User Goal:** Récupérer le lien de la PR.

**URL/Route:** `/create/success`

---

## Layout Sections

### 1. Success Message
- **Purpose:** Confirmation visuelle forte.
- **Priority:** Primary.
- **Objects:**
    - `success-icon`: Icône de sceau (Seal) vert brillant.
    - `success-title`: "Secret Scellé avec Succès !"

### 2. PR Link Section
- **Purpose:** Action principale de sortie.
- **Priority:** Primary.
- **Objects:**
    - `pr-link-card`: Carte affichant le numéro de la PR et un bouton "Ouvrir dans GitHub".

---

## Content & Languages

- **Body Text:**
    - FR: "Votre secret a été chiffré et poussé vers la branche `feature/secret-mes-db`. Une Pull Request est prête pour validation."
    - EN: "Your secret has been encrypted and pushed to branch `feature/secret-mes-db`. A Pull Request is ready for review."

---

## Interactions & States

- **`btn-open-github`:** Ouvre un nouvel onglet vers l'URL de la PR.
