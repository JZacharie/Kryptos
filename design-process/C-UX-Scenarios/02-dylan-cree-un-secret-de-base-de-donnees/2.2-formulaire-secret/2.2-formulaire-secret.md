# 2.2 Formulaire de Configuration

**Scenario:** Dylan crée un secret de base de données
**Page Number:** 2.2
**Created:** 2026-04-24
**Method:** Whiteport Design Studio (WDS)

---

## Overview

**Page Purpose:** Collecter les métadonnées nécessaires pour générer le SealedSecret (Nom, Namespace cible, Chemin Vault).

**User Mental State:**
Prudence technique. Dylan veut s'assurer que le chemin Vault (`sanofi/kv/...`) est le bon. Il apprécie l'auto-complétion.

**Main User Goal:** Remplir le formulaire sans erreur de validation.

**URL/Route:** `/create/form`

---

## Layout Sections

### 1. Form Section
- **Purpose:** Champs de saisie structurés.
- **Priority:** Primary.
- **Objects:**
    - `input-ns`: Sélecteur de Namespace (mes-prod, mes-staging).
    - `input-secret-name`: Nom du secret (ex: mes-db-credentials).
    - `input-vault-path`: Chemin source dans HashiCorp Vault.
    - `input-rotation`: Toggle pour la rotation automatique (Optionnel).

### 2. Live Preview (Sidebar)
- **Purpose:** Montrer à quoi ressemblera le fichier YAML généré en temps réel.
- **Priority:** Secondary (Power user feature).
- **Objects:**
    - `yaml-preview-box`: Code block avec coloration syntaxique.

---

## Content & Languages

- **Labels:**
    - FR: "Nom du secret Kubernetes", "Chemin Vault source"
    - EN: "Kubernetes Secret Name", "Source Vault Path"
- **Placeholders:**
    - FR: "ex: my-app-secret", "ex: team-a/db/prod"

---

## Validation Rules

- **Secret Name:** Doit être en minuscule, sans espaces, caractères speciaux autorisés : `-`.
- **Vault Path:** Doit commencer par le mount point `sanofi/`.
