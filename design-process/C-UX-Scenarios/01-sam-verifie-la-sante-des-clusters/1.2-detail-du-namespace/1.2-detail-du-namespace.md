# 1.2 Détail du Namespace (mes-prod)

**Scenario:** Sam vérifie la santé des clusters
**Page Number:** 1.2
**Created:** 2026-04-24
**Method:** Whiteport Design Studio (WDS)

---

## Overview

**Page Purpose:** Inspecter précisément la liste des secrets d'un namespace spécifique et déclencher une action corrective si nécessaire.

**Entry Points:**
- Clic sur une carte Namespace depuis le Dashboard (1.1).
- Recherche directe via la navigation globale.

**User Mental State:**
Investigation focalisée. Sam veut savoir *quel* secret est en alerte et *pourquoi*. Il a besoin d'informations précises : date d'expiration exacte dans Vault vs version actuelle dans Git.

**Main User Goal:** Identifier le secret fautif et confirmer la date d'expiration.

**Business Goal:** Transparence technique totale pour accélérer la résolution d'incidents.

**URL/Route:** `/namespace/{id}` (ex: `/namespace/mes-prod`)

## Layout Sections

### 1. Breadcrumb & Actions
- **Purpose:** Navigation de retour et actions de haut niveau sur le namespace.
- **Priority:** Secondary.
- **Objects:**
    - `nav-breadcrumb`: Dashboard > mes-prod.
    - `btn-sync-git`: Bouton "Force Git Sync" (Ghost style).

### 2. Namespace Header (Identity)
- **Purpose:** Afficher le nom du namespace et son état de santé global.
- **Priority:** Primary.
- **Objects:**
    - `ns-title`: Nom du namespace (heading-4xl).
    - `ns-status-badge`: Badge large avec icône d'état (Healthy/Warning/Error).

### 3. Secrets Inventory (Table)
- **Purpose:** Liste exhaustive des secrets synchronisés.
- **Priority:** Primary.
- **Objects:**
    - `secrets-table`: Tableau technique (Nom, Source Vault, Etat Git, Expiration).
    - `expiration-date-tag`: Tag coloré selon l'urgence.

---

## Content & Languages

- **Status Labels:**
    - FR: "Sain", "Attention", "Critique"
    - EN: "Healthy", "Warning", "Critical"
- **Table Headers:**
    - FR: "Nom du Secret", "Dernière mise à jour", "Expire le"
    - EN: "Secret Name", "Last Update", "Expires on"

---

## Interactions & States

- **`btn-sync-git`:**
    - **Click:** Déclenche une demande de webhook vers GitOps. État "Syncing..." avec spinner discret.
- **`expiration-date-tag`:**
    - **Hover:** Affiche une infobulle (Tooltip) avec le chemin exact dans HashiCorp Vault.
